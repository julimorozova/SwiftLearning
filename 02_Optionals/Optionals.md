# Опицональные типы

``` swift
var name: String?  // nil
```

nil == empty пустота, ничего не пришло
 
Int != Int?
это разные типы данных, потому что Int? содержит в себе еще nil
опциональные типы нужно разворачивать  

Эти переменные нужны тогда, когда данные могут не прийти, может быть ошибка и тд

Математические операции не применимы к опционалам напрямую без извлечений 

### forceunwrap - принудительное извлечение 

``` swift 
var name: String?
print(name!) 
```
если там пустота, то будет ошибка и программа рухнет, так лучше не делать 
можно написать только в случае если 100% уверены, но это риски

### optional binding 

``` swift 
if let data {
    print(data)
}
if data != nil {
    data = nil // такое есть риск быть, в первом случае такой сценарий не возможен, так как там создается новая костанта и она уже не может быть переопреденена в nil, и она создастся только есть data не nil
    print(data)
}
```
